Core Features
2.1. Module: User & Member Management
2.1.1. JWT/Sanctum Authentication

User Registration (Mobile & Email verification via OTP).

Login/Logout functionality.

Password reset flow.

Token-based authentication for all subsequent API requests.
2.1.2. Comprehensive Profile Management

View and edit personal details (Name, Contact, Email).

Profile picture upload (integration with cloud storage like S3 or DigitalOcean Spaces).

Digital Membership ID Card: A dedicated screen within the app displaying a QR code and member details. This QR code can be scanned at club entry or for verification.
2.1.3. History & Tracking

Paginated list of all past and upcoming facility bookings.

Order history for F&B purchases.

Payment transaction history with filters.
2.2. Module: Facility & Event Management (Admin-Focused)
2.2.1. Facility Management (Admin Dashboard)

CRUD (Create, Read, Update, Delete) operations for facilities (e.g., Tennis Court 1, Olympic Pool, Squash Court).

Define operating hours, slot duration (e.g., 60 mins), and maintenance schedules.
2.2.2. Event Management (Admin Dashboard)

Create events with title, description, image, date, time, capacity, and fee.

Publish/Unpublish events to the mobile app and website.

View list of members registered for an event.
2.3. Module: Game Booking System
2.3.1. Discovery & Availability

User selects a date and facility type.

The app displays available time slots in a grid UI.

Real-time Availability API: A dedicated, optimized API endpoint that queries the MS SQL Server database to check member status and the MySQL bookings table to check for conflicts, returning available slots in real-time.
2.3.2. Booking Flow

User selects an available slot.

System calculates charge (if any) based on member type and time.

User proceeds to a secure Razorpay payment page (if payment is required).

Upon successful payment, a booking is created, and a confirmation notification is sent.
2.3.3. Booking Modifications

Users can view "My Bookings".

Cancellation allowed based on a predefined cancellation policy (e.g., free cancellation up to 12 hours prior). Refunds are processed via Razorpay if applicable.
2.4. Module: Food & Beverage (F&B)
2.4.1. Dynamic Menu

Admin can manage menu categories and items (name, description, price, image, availability status) from the dashboard.

The app fetches the active menu via API.
2.4.2. Ordering System

Users add items to a cart.

Checkout process with order summary.

Integration with Razorpay for payment.

Order confirmation sent to the kitchen/bar system (via email/API) and user.
2.5. Module: Payment Integration (Razorpay)
2.5.1. Payment Processing

Secure collection of payment details via Razorpay's checkout modal.

Handling of successful and failed payments.
2.5.2. Webhook & Callback Handling

A secure webhook endpoint in Laravel to receive payment status updates from Razorpay (e.g., payment.captured, payment.failed). This ensures database records are updated even if the user closes the app mid-process.
2.5.3. Automated Invoice Generation

Upon successful payment, a PDF invoice is automatically generated (using libraries like laravel-dompdf).

Invoices are stored against the user's record and are downloadable from the "History" section.

Advanced Integrations & Special Features
3.1. WhatsApp Business API Integration

Use Cases:
Booking confirmations and reminders (24h before slot).
Payment receipts and invoice links.
Event reminders and updates.
Broadcast messages for news/offers (opt-in required).
3.2. Legacy System Integration (.NET & MS SQL)

Develop a dedicated set of read-only APIs within the Laravel backend to communicate with the existing MS SQL Server database.

Purpose: To validate active membership status and check real-time booking locks from the legacy system before confirming a new booking in the primary MySQL database.
3.3. Monthly Billing Automation

Implement Laravel Scheduled Commands (Cron Jobs).

On a specified day each month, the system will:

Identify members with auto-debit subscriptions.

Generate invoices for fixed monthly charges.

Trigger automatic payment collection via Razorpay's Subscription or Tokenization features.

Send payment success/failure notifications via Email/WhatsApp.
3.4. News & Updates Management (CRUD)

Admin dashboard section to create and publish news articles and updates.

Articles are displayed in a dedicated "News Feed" within the mobile app.

Option to send push notifications for critical announcements.
